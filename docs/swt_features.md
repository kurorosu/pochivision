# SWT 特徴量ガイド

SWT (Stationary Wavelet Transform) 特徴量抽出器の各特徴量とパラメータの解説.

## 概要

SWT は画像をスケール (粗さ) ごとに分解し, 各スケールの方向性エネルギーやテクスチャ特性を定量化する. FFT が画像全体の周波数成分を見るのに対し, SWT は方向別 (水平エッジ, 垂直エッジ, 対角) のテクスチャを階層的に分析できる.

### 処理フロー

1. グレースケール変換
2. リサイズ (`resize_shape` 設定時)
3. SWT 用サイズ調整 (2^max_level の倍数にパディング)
4. [0, 1] に正規化 (`max > 1.0` の場合 `/255`)
5. `pywt.swt2` でウェーブレット分解
6. 各レベル・各サブバンドの統計量を計算

### サブバンドの構成

SWT は各レベルで画像を 4 つのサブバンドに分解する:

```
        低域フィルタ(行)    高域フィルタ(行)
低域フィルタ(列)  LL (近似)          HL (水平エッジ検出)
高域フィルタ(列)  LH (垂直エッジ検出)  HH (対角エッジ検出)
```

- **LL**: 低周波近似. 元画像のぼやけたバージョン
- **LH**: 垂直エッジを検出 (水平ストライプに反応)
- **HL**: 水平エッジを検出 (垂直ストライプに反応)
- **HH**: 対角エッジを検出

注意: LH は「垂直エッジ」を検出するため, **水平**ストライプに強く反応する (エッジ方向とストライプ方向は直交).

## 特徴量一覧

各レベルで以下の 20 特徴量が抽出される.
マルチスケール時の特徴量名: `L{level}_{stat}_{subband}` (例: `L1_energy_lh`)

### 基本統計量 (各サブバンド x 4)

| 特徴量 | 単位 | 説明 |
|--------|------|------|
| `mean_{ll,lh,hl,hh}` | coefficient | 係数の平均値 |
| `energy_{ll,lh,hl,hh}` | coefficient_squared | 係数の二乗和. テクスチャの強さを表す |
| `entropy_{ll,lh,hl,hh}` | bits | シャノンエントロピー. 係数分布の複雑さ |
| `std_{ll,lh,hl,hh}` | coefficient | 係数の標準偏差 |

### エネルギー比 (方向別)

| 特徴量 | 単位 | 値の範囲 | 説明 |
|--------|------|---------|------|
| `energy_ratio_h` | ratio | 0.0~1.0 | LH エネルギー / 全 detail エネルギー. 垂直エッジの割合 |
| `energy_ratio_v` | ratio | 0.0~1.0 | HL エネルギー / 全 detail エネルギー. 水平エッジの割合 |
| `energy_ratio_d` | ratio | 0.0~1.0 | HH エネルギー / 全 detail エネルギー. 対角エッジの割合 |
| `total_energy` | coefficient_squared | 0.0~ | LL + LH + HL + HH の全エネルギー |

3 つの ratio の合計は ~1.0.

## 画像例との対応 (L1, wavelet=db1)

| 画像 | energy_lh | energy_hl | energy_hh | ratio_h | ratio_v | ratio_d |
|------|-----------|-----------|-----------|---------|---------|---------|
| 均一画像 | 0 | 0 | 0 | 0.0 | 0.0 | 0.0 |
| 水平ストライプ | 2048 | 0 | 0 | 1.0 | 0.0 | 0.0 |
| 垂直ストライプ | 0 | 2048 | 0 | 0.0 | 1.0 | 0.0 |
| ランダム | 337 | 351 | 366 | 0.32 | 0.33 | 0.35 |

**水平ストライプ**: LH (垂直エッジ検出) のエネルギーが 100%, HL/HH はゼロ.
**垂直ストライプ**: HL (水平エッジ検出) のエネルギーが 100%.
**ランダム**: 全方向ほぼ均等 (~1/3 ずつ).

### 方向の対称性

水平ストライプの LH エネルギー = 垂直ストライプの HL エネルギー (同じ値で入れ替わる).

## マルチスケール

`max_level > 1` の場合, 各レベルの特徴量が `L1_`, `L2_`, ... のプレフィックスで出力される.

```
max_level=1: L1_ のみ (20 特徴量)
max_level=2: L1_ + L2_ (40 特徴量)
max_level=3: L1_ + L2_ + L3_ (60 特徴量)
```

- **L1** = 最細 (高周波, 細かいテクスチャ)
- **LN** = 最粗 (低周波, 大まかなパターン)

`multiscale=False` の場合は L1 (高周波) のみ, プレフィックスなし.

## パラメータ

| パラメータ | デフォルト | 説明 |
|-----------|----------|------|
| `wavelet` | `"db1"` | ウェーブレット関数. `db1`(Haar), `db4`, `sym4` 等 |
| `max_level` | `1` | 分解レベル数. 高いほど粗いスケールまで分解 |
| `multiscale` | `true` | 全レベルの特徴量を出力するか. false なら L1 のみ |
| `resize_shape` | `null` | リサイズ形状. `[512, 512]` 等. null=リサイズしない |
| `preserve_aspect_ratio` | `true` | アスペクト比を保持するか |
| `aspect_ratio_mode` | `"width"` | アスペクト比保持時の基準軸 |

### `wavelet` の選択

| ウェーブレット | 特徴 |
|--------------|------|
| `db1` (Haar) | 最もシンプル. ステップ状のエッジに適する |
| `db4` | 滑らかな波形. テクスチャのグラデーションに適する |
| `sym4` | 対称性が高い. アーティファクトが少ない |

### `max_level` の目安

```
max_level=1: 1px 単位の細かいエッジ (20 特徴量)
max_level=2: 2px 単位の中程度のパターンも捕捉 (40 特徴量)
max_level=3: 4px 単位の粗いパターンも捕捉 (60 特徴量)
```

画像サイズの制約: 各辺が `2^max_level` の倍数である必要がある (不足分は自動パディング).

## 設計上の制約

1. **最小画像サイズ**: 4x4 未満の画像は `ValueError` を送出する.
2. **サイズ調整**: `2^max_level` の倍数になるようエッジリプリケーションでパディング. パディング量が大きい場合 (高 level + 小画像), 境界アーティファクトが生じる可能性がある.
3. **dtype 正規化**: `max > 1.0` の場合 `/255` で [0, 1] に正規化. uint8 と float32 (0-255) で同じ特徴量が得られる.
4. **レベルラベリング**: L1=高周波 (最細), LN=低周波 (最粗). 標準ウェーブレット慣習に準拠.
5. **`get_feature_names` の config 依存**: 特徴量名は `multiscale` と `max_level` に依存する. デフォルト config と異なる設定を使う場合は config を渡す必要がある.

## 設定例

```json
{
    "swt": {
        "wavelet": "db4",
        "max_level": 1,
        "multiscale": true,
        "resize_shape": [512, 512],
        "preserve_aspect_ratio": true,
        "aspect_ratio_mode": "width"
    }
}
```
