# LBP 特徴量ガイド

LBP (Local Binary Pattern) 特徴量抽出器の各特徴量とパラメータの解説.

## 概要

LBP は各ピクセルの周囲パターンを 0/1 の二値コードに変換し, そのパターンの出現分布でテクスチャを定量化する. 計算が高速でグレースケール不変性を持つ.

### 処理フロー

1. グレースケール変換
2. リサイズ (`resize_shape` 設定時)
3. `skimage.feature.local_binary_pattern` でLBP画像を計算
4. メソッド別のビン数・範囲でヒストグラムを計算
5. ヒストグラムを手動正規化 (和=1)
6. 統計量を計算

### LBP パターンの仕組み

各ピクセルについて, 半径 R の円上に P 個の近傍点を配置し, 中心と比較:

```
P=8, R=1 の場合:

  7  0  1      近傍 >= 中心 → 1
  6  C  2      近傍 <  中心 → 0
  5  4  3

例: 中心=100, 近傍=[120, 90, 110, 80, 130, 95, 105, 85]
    → パターン: [1, 0, 1, 0, 1, 0, 1, 0] → 非uniform
```

## 特徴量一覧

### 統計量 (LBP 画像から直接計算)

| 特徴量 | 単位 | 説明 |
|--------|------|------|
| `lbp_mean` | lbp_value | LBP 画像の平均値. テクスチャの粗さとの相関は間接的 |
| `lbp_std` | lbp_value | LBP 画像の標準偏差. 大きいほどパターンが多様 |
| `lbp_skewness` | dimensionless | LBP 値の歪度 |
| `lbp_kurtosis` | dimensionless | LBP 値の尖度 |

### ヒストグラムベース統計量

| 特徴量 | 単位 | 値の範囲 | 説明 |
|--------|------|---------|------|
| `lbp_entropy` | normalized | 0.0~1.0 | 正規化エントロピー. パターンの多様性. 高いほど多様 |
| `lbp_energy` | ratio | 0.0~1.0 | エネルギー (sum(p^2)). パターンの集中度. 高いほど少数パターンに集中 |

### ヒストグラム (`include_histogram=true` 時)

| 特徴量 | 単位 | 説明 |
|--------|------|------|
| `lbp_bin_0` ~ `lbp_bin_9` | ratio | 各パターンの出現比率 (uniform, P=8 の場合 10 ビン) |

uniform メソッド (P=8) のビンの意味:

| ビン | パターン | 意味 |
|------|---------|------|
| bin_0 | 00000000 | 周囲すべてが中心より暗い |
| bin_1 | 00000001 | 1箇所だけ明るい |
| bin_2 | 00000011 | 連続2箇所が明るい |
| ... | ... | ... |
| bin_8 | 11111111 | 周囲すべてが中心より明るい |
| bin_9 | non-uniform | 0→1の遷移が3回以上の不規則パターン |

## 画像例との対応 (uniform, P=8, R=1)

| 画像 | entropy | energy | bin_0 | bin_8 | bin_9 | 特徴 |
|------|---------|--------|-------|-------|-------|------|
| 全黒 | 0.00 | 1.00 | 0.00 | 1.00 | 0.00 | 全ピクセルが同一パターン |
| 均一 | 0.10 | 0.88 | 0.00 | 0.94 | 0.00 | ほぼ全ピクセルが bin_8 |
| チェッカーボード | 0.30 | 0.50 | 0.50 | 0.50 | 0.00 | 2パターンが均等 |
| 水平ストライプ | 0.33 | 0.48 | 0.00 | 0.50 | 0.00 | 2~3パターン |
| ランダム | 0.88 | 0.16 | 0.18 | 0.18 | 0.24 | 全パターンに分散, non-uniform が多い |

**注意**: uniform LBP は回転不変のため, 水平ストライプと垂直ストライプは同じ特徴量を返す.

## メソッド別のビン数

| method | ビン数 (P=8) | 特徴 |
|--------|-------------|------|
| `uniform` | P + 2 = 10 | 回転不変 + uniform. 最もコンパクト |
| `nri_uniform` | P(P-1) + 3 = 59 | uniform だが回転を区別. 方向性を保持 |
| `var` | 256 | 局所分散の連続値. テクスチャの粗さを直接測定 |
| `default` | 2^P = 256 | 全パターン. 最も高次元 |
| `ror` | - | 回転不変 (uniform制約なし) |

## パラメータ

| パラメータ | デフォルト | 説明 |
|-----------|----------|------|
| `P` | `8` | 近傍点数. 多いほど詳細なパターンを捉えるが次元が増加 |
| `R` | `1` | 半径 (ピクセル). 大きいほど粗いテクスチャを捉える |
| `method` | `"uniform"` | LBP 手法. `uniform`, `nri_uniform`, `var`, `default`, `ror` |
| `resize_shape` | `[128, 128]` | リサイズ形状. null=リサイズしない |
| `include_histogram` | `false` | ヒストグラムの各ビンを特徴量に含むか |
| `preserve_aspect_ratio` | `true` | アスペクト比を保持するか |
| `aspect_ratio_mode` | `"width"` | アスペクト比保持時の基準軸 |

### `P` と `R` の調整

```
P=8,  R=1: 1px 半径の 8 近傍 (微細テクスチャ)
P=16, R=2: 2px 半径の 16 近傍 (中程度テクスチャ)
P=24, R=3: 3px 半径の 24 近傍 (粗いテクスチャ)
```

### `method` の選択

```
uniform:     回転不変 + コンパクト (10 ビン) → 一般的な用途に推奨
nri_uniform: 方向性を保持 (59 ビン) → 方向性のあるテクスチャの区別に
var:         局所分散 (256 ビン) → テクスチャの粗さを直接測定
default:     全パターン (256 ビン) → 最も詳細だが高次元
```

### `include_histogram` の効果

```
false: 6 特徴量 (mean, std, skewness, kurtosis, entropy, energy)
true:  6 + ビン数 の特徴量 (uniform なら 16 特徴量)
```

ヒストグラムは各パターンの出現比率を直接示すため, 最も情報量が多い特徴量.

## 特徴量の有効度

| 有効度 | 特徴量 | 理由 |
|--------|--------|------|
| 高 | `entropy`, `energy` | パターン分布の形状を直接反映. 解釈が明確 |
| 高 | `histogram` (bin_0~9) | 各パターンの出現比率. 最も情報量が多い |
| 中 | `std` | パターンの多様性の間接指標. 弁別力はある |
| 低 | `mean`, `skewness`, `kurtosis` | パターン番号の統計. 弁別力はあるが解釈困難 |

## 設計上の制約

1. **グレースケール不変**: 近傍と中心の大小比較のため, 画像全体の明るさに非依存.
2. **回転不変性**: `uniform` メソッドでは回転を同一視. 水平/垂直ストライプが同じ値になる.
3. **ヒストグラム正規化**: `density=False` + 手動正規化 (和=1) で確率分布を計算.
4. **エントロピー正規化**: `log2(n_bins)` で割り [0, 1] 範囲. FFT/SWT と統一.

## 設定例

```json
{
    "lbp": {
        "P": 8,
        "R": 1,
        "method": "uniform",
        "resize_shape": [512, 512],
        "include_histogram": true,
        "preserve_aspect_ratio": true,
        "aspect_ratio_mode": "width"
    }
}
```
