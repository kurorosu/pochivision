# FFT 特徴量ガイド

FFT (高速フーリエ変換) 特徴量抽出器の各特徴量とパラメータの解説.

## 概要

画像を周波数領域に変換し, テクスチャや周期性を定量化する. 画像の「滑らかさ」「繰り返しパターン」「方向性」を数値化できる.

### 前処理

1. グレースケール変換
2. `float64` に変換 (コントラスト情報を保持)
3. **Hanning 窓関数**を適用 (画像境界のスペクトルリークを抑制)
4. 2D FFT を実行
5. **DC 成分** (平均輝度) を除外し, AC 成分のみで特徴量を計算

## 特徴量一覧

### エネルギー系

| 特徴量 | 単位 | 値の範囲 | 説明 |
|--------|------|---------|------|
| `high_low_ratio` | ratio | 0.0〜 | 高周波エネルギー / 低周波エネルギー. 大きいほどテクスチャが多い |
| `band_1_0.00_0.10` | ratio | 0.0〜1.0 | 低周波帯エネルギー比率 |
| `band_2_0.10_0.30` | ratio | 0.0〜1.0 | 中周波帯エネルギー比率 |
| `band_3_0.30_0.50` | ratio | 0.0〜1.0 | 高周波帯エネルギー比率 (非正方形画像では対角線の超過分も含む) |

3つの帯域エネルギーの合計は ~1.0 になる.

```
グラデーション画像:   band_1 ≈ 1.0, band_2 ≈ 0.0, band_3 ≈ 0.0
ストライプ画像:       band_1 ≈ 0.4, band_2 ≈ 0.6, band_3 ≈ 0.0
ランダムノイズ:       band_1 ≈ 0.6, band_2 ≈ 0.1, band_3 ≈ 0.2
```

### 方向性

| 特徴量 | 単位 | 値の範囲 | 説明 |
|--------|------|---------|------|
| `horizontal_energy` | ratio | 0.0〜1.0 | 水平方向の周波数エネルギー比率. 垂直ストライプで高い |
| `vertical_energy` | ratio | 0.0〜1.0 | 垂直方向の周波数エネルギー比率. 水平ストライプで高い |

FFT では画像の空間パターンと周波数方向が直交する:
- **水平ストライプ** → **垂直方向**の周波数成分 → `vertical_energy` が高い
- **垂直ストライプ** → **水平方向**の周波数成分 → `horizontal_energy` が高い

```
水平ストライプ:   h_energy ≈ 0.14, v_energy ≈ 0.78
垂直ストライプ:   h_energy ≈ 0.78, v_energy ≈ 0.14
```

### スペクトル統計

| 特徴量 | 単位 | 値の範囲 | 説明 |
|--------|------|---------|------|
| `spectral_centroid` | cycle/pixel or cycle/mm | 0.0〜0.5 | スペクトルの重心. 大きいほど高周波寄りのテクスチャ |
| `spectral_std` | cycle/pixel or cycle/mm | 0.0〜 | スペクトルの広がり. 大きいほど周波数成分が分散 |

```
グラデーション:   centroid ≈ 0.01, std ≈ 0.01 (低周波に集中)
ストライプ:       centroid ≈ 0.17, std ≈ 0.11 (中周波にピーク, 窓関数で若干広がる)
ランダムノイズ:   centroid ≈ 0.36, std ≈ 0.13 (全帯域に分散)
```

`mm_per_pixel` が設定されている場合, 単位は cycle/mm に変換される.

### ピーク検出

| 特徴量 | 単位 | 値の範囲 | 説明 |
|--------|------|---------|------|
| `num_peaks` | count | 0〜 | 支配的な周波数成分の数. 多いほど複雑なテクスチャ |
| `max_peak_amp` | amplitude | 0.0〜 | 検出されたピークの中の最大振幅 |

```
スペクトルを山の地形に例えると:

振幅
 |     ★ 最も高い山 (100)
 |     /\
 |    /  \   ★ 中くらいの山 (30) → ピーク (30 > 閾値 10)
 |   /    \ /\
 |  /      v  \    ★ 小さい丘 (5) → 無視 (5 < 閾値 10)
 | /            \  /\
 |/              \/  \___
 +------------------------→ 周波数

peak_threshold_ratio = 0.1 の場合:
  閾値 = 最大値 100 × 0.1 = 10
  → num_peaks = 2 (最大値の山 + 中くらいの山)
```

### エントロピー

| 特徴量 | 単位 | 値の範囲 | 説明 |
|--------|------|---------|------|
| `spectral_entropy` | normalized | 0.0〜1.0 | スペクトル全体のエントロピー. 高いほど周波数成分が均等に分散 |
| `horizontal_entropy` | normalized | 0.0〜1.0 | 水平方向のスペクトルエントロピー |
| `vertical_entropy` | normalized | 0.0〜1.0 | 垂直方向のスペクトルエントロピー |
| `band_1_0.00_0.10_entropy` | normalized | 0.0〜1.0 | 低周波帯のエントロピー |
| `band_2_0.10_0.30_entropy` | normalized | 0.0〜1.0 | 中周波帯のエントロピー |
| `band_3_0.30_0.50_entropy` | normalized | 0.0〜1.0 | 高周波帯のエントロピー |

Shannon エントロピーを `log2(N)` で正規化しており, 異なるピクセル数の領域間で直接比較可能.

```
単一周波数:   entropy ≈ 0.3 (エネルギーが1箇所に集中)
ランダム:     entropy ≈ 0.97 (エネルギーが均等に分散)
```

## パラメータ

| パラメータ | デフォルト | 説明 |
|-----------|----------|------|
| `frequency_bands` | `[[0.0, 0.1], [0.1, 0.3], [0.3, 0.5]]` | 帯域エネルギーの周波数範囲. 最終帯域は上限なし (対角線の超過分を含む) |
| `high_low_threshold` | `0.2` | 低周波/高周波の境界. `high_low_ratio` の計算に使用 |
| `directional_tolerance` | `10.0` | 方向性エネルギーの許容角度 (度). 例: 0° ± 10° を「水平方向」とみなす |
| `peak_threshold_ratio` | `0.1` | ピーク検出の閾値. スペクトル最大値に対する比率. 小さいほど多くのピークを検出 |
| `mm_per_pixel` | `null` | 1ピクセルあたりの物理サイズ (mm). 設定すると `spectral_std` と `spectral_centroid` が cycle/mm 単位に変換される |

### `frequency_bands` の調整例

```json
// デフォルト (低/中/高の3帯域)
"frequency_bands": [[0.0, 0.1], [0.1, 0.3], [0.3, 0.5]]

// より細かい帯域分割
"frequency_bands": [[0.0, 0.05], [0.05, 0.1], [0.1, 0.2], [0.2, 0.3], [0.3, 0.5]]

// 低/高の2帯域のみ
"frequency_bands": [[0.0, 0.2], [0.2, 0.5]]
```

### `directional_tolerance` の図

```
     90° (垂直)
      |
      | ← ±10° の範囲が "垂直"
      |
0° ---+--- 180° (水平)  ← ±10° の範囲が "水平"
      |

tolerance=10.0: 狭い方向性 (厳密な方向検出)
tolerance=30.0: 広い方向性 (大まかな方向検出)
```

## 設計上の制約

1. **Hanning 窓**: 画像端のピクセルがゼロに減衰するため, 端に集中する情報は抑制される. `max_peak_amp` の絶対値はピクセル位置に依存する.
2. **最小画像サイズ**: 4x4 未満の画像は `ValueError` を送出する. Hanning 窓が有効に機能するには最低 4x4 が必要.
3. **非正方形画像**: `freq_norm` は `max(cx, cy)` で正規化するため, 短辺方向の Nyquist 周波数は 0.5 に達しない. 最終帯域が上限なしで対角線分を含めることで帯域合計 ~1.0 を保証.

## 設定例

```json
{
    "fft": {
        "frequency_bands": [
            [0.0, 0.1],
            [0.1, 0.3],
            [0.3, 0.5]
        ],
        "high_low_threshold": 0.2,
        "directional_tolerance": 10.0,
        "peak_threshold_ratio": 0.1,
        "mm_per_pixel": 0.05
    }
}
```
